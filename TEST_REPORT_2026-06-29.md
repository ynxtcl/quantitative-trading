## 6. ⚠️ Phase 4: XGBoost AI (`main_phase4.py`)

### 问题：完整运行时 segfault (exit code 3221225477)

**根因分析**: STATUS_ACCESS_VIOLATION 发生于多标的 × 多轮 × 108 组合网格搜索场景下，推断为：

1. **内存耗尽**: `main_phase4.py` 对 3 只股票各跑最多 3 轮 WF，每轮执行 108 个 XGBoost 模型（网格搜索），`n_jobs=-1`（全部 CPU 核）导致大并发内存超额
2. **Python 3.13 兼容性**: 该版本较新，xgboost 3.3.0 的 C 扩展可能有内存管理的边缘 case
