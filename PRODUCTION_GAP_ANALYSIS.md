# 生产环境差距分析 — 从回测原型到实盘交易

> 分析日期: 2026-06-29 | 分析范围: 全代码库 17+ 核心模块
> 
> **总体评分: 3/10** — 优秀的学术级研究原型，距实盘交易仍有较大距离

---

## 1. 数据层 — 关键阻塞项

### 1.1 日频硬编码 (Confidence 10/10)
`config/settings.py` L37: `"freq": "1d"`。全系统仅支持日线。实盘交易需要分钟级数据。

### 1.2 数据源不可靠 (Confidence 10/10)
`data/loader.py` L158: 唯一数据源是免费的 `akshare`。
- 无 SLA，接口频繁变动（文档说 1-2 天修复）
- 无实时数据 — 盘后才能获取
- 无 WebSocket/推送机制
- 生产需要：Wind/Choice/tushare pro（均付费）

### 1.3 无实时数据管道 (Confidence 10/10)
零实时基础设施。无消息队列，无流处理。

### 1.4 无交易日历处理 (Confidence 9/10)
`data/cleaner.py` L84: 使用 `freq='B'` 但不处理：
- A 股节假日（春节、国庆等）
- 半日交易
- T+1 结算规则
- 涨跌停板

### 1.5 容错机制为零 (Confidence 10/10)
`_fetch_from_akshare` 用 `except Exception` 统一捕获，返回空 DF。
生产需要：指数退避重试、数据源故障切换、数据完整性校验、监控告警。

### 1.6 无数据版本管理 (Confidence 9/10)
Parquet 缓存会被直接覆盖。无数据血缘追溯、无复现性保证。

## 2. 执行层 — 根本性缺失

### 2.1 无券商对接 (Confidence 10/10)
零券商 API 集成。A 股零售量化接口：华泰(xtquant)、国信、招商，均需资产门槛和纸质合同。

### 2.2 `BacktestEngine._execute_signal` 假设立即成交 (Confidence 10/10)
`engine.py` L168: `price * (1 + slippage)`。现实中：
- 市价单：成交价不确定
- 限价单：可能不成交
- 部分成交存在
- A 股 T+1：今日买入不可卖出
- 无订单状态跟踪、无撤单、无改单

### 2.3 无订单管理系统 (Confidence 10/10)
缺失：订单生命周期、订单簿管理、超时重提、TWAP/VWAP/POV 算法。

### 2.4 `int()` 截断导致系统性偏差 (Confidence 10/10)
`engine.py` L173: `max_qty = int(...)` 始终**向下取整**。10万账户 × 30% → ¥30,000 → 买 ~1300 股 ¥22 股票，每次损失 ¥11.72。800+笔交易累积 **¥9,400**（初始资本的 10%）在沉默中丢失。

### 2.5 `max_qty` 假设 1 股起买 (Confidence 9/10)
A 股最小交易单位是 1 手 = 100 股。应强制 `max_qty = (max_qty // 100) * 100`。

### 2.6 忽略"最低 5 元"佣金规则 (Confidence 9/10)
`settings.py` L30: `"commission": 0.0003` 但 A 股有 `max(5, 金额×0.0003)`。对小账户，实际佣金翻倍。

### 2.7 有止损无止盈 (Confidence 9/10)
`portfolio/engine.py` L248-256: 仅有 `_check_stop_loss()`。无：移动止损、目标止盈、时间止损、波动率自适应止盈。

## 3. 风控层 — 生产环境差距

### 3.1 无交易前风控 (Confidence 10/10)
`RiskManager.filter_signals()` 在信号生成之后、执行之前运行。缺失：
- 合规检查（黑名单、交易限制）
- 资金充足性检查（考虑挂单中的资金）
- 跨市场风险检查
- 实时仓位限制

### 3.2 回撤熔断无紧急平仓逻辑 (Confidence 9/10)
`risk_manager.py` L93: 回撤超标时返回空信号，但**持仓仍然存在**。不应禁止开仓，应强制平仓。

### 3.3 无投资组合 VaR/CVaR (Confidence 9/10)
无在险价值、条件在险价值、压力测试、蒙特卡洛模拟。

### 3.4 未跟踪相关性风险 (Confidence 9/10)
仅做多沪深 300 成分股。大盘崩溃时所有持仓正相关，分散化失效。

## 4. 策略层 — 生产环境短板

### 4.1 趋势跟踪: `exit_period=10` 存在未来信息 (Confidence 9/10)
`strategy_config.py` L28: 10 日低点包含**当日最低价** 。卖出信号使用今日价格决定今日卖出 — 这是微妙但真实的未来信息泄露。

### 4.2 FactorSelectionStrategy 无法用于单股 (Confidence 10/10)
`strategy.py` L148: `data.nlargest(...)` 需要多股票合并的 DataFrame。该策略类在当前架构中实为**死代码**。

### 4.3 无市场状态识别 (Confidence 10/10)
所有策略在牛市、熊市、震荡市中使用相同参数。生产需要：状态分类器、状态依赖参数切换、策略权重调整。

### 4.4 XGBoost 模型漂移未检查 (Confidence 9/10)
`model_trainer.py` — Walk-Forward 重训练但：
- 无概念漂移检测
- 无模型质量门禁（验证准确率低于阈值时跳过交易）
- 无特征重要性稳定性监控
- 无模型版本管理/回滚

## 5. 运维层 — 完全缺失

### 5.1 无日志基础设施 (Confidence 10/10)
零结构化日志：无每日交易日志、无错误聚合、无性能追踪。所有输出依赖 `print()`。

### 5.2 无监控告警 (Confidence 10/10)
无：健康检查端点、策略仪表盘、盈亏对比、仓位对账、微信/邮件/短信告警。

### 5.3 无定时执行 (Confidence 10/10)
`main.py`、`main_portfolio.py`、`main_phase4.py` 均为手动脚本。无：
- 日频调度器（cron / 任务计划程序）
- 交易时段控制（A 股：9:30-11:30, 13:00-15:00）
- 收盘信号生成 → 次日开盘提交

### 5.4 无数据库 (Confidence 10/10)
除 parquet 文件外无任何持久化：
- 无 SQL 数据库存储交易历史
- 无时序数据库存储市场数据
- 无状态持久化（进程崩溃后无法恢复）
- 进程死在中途 = 丢失全部状态

### 5.5 无配置管理 (Confidence 9/10)
`config/settings.py` 硬编码。生产需要：环境配置（dev/staging/prod）、密钥管理、动态热更新。

### 5.6 无比对验证 (Confidence 9/10)
`verify/` 目录的 rqalpha 对比已损坏。无：实盘 Walk-Forward 追踪、夏普衰减监控、滑点影响分析。

## 6. 架构层 — 结构性问题

### 6.1 Python GIL 限制性能 (Confidence 9/10)
全系统为单线程 Python。100+ 股票 × 多策略 × ML 模型推理 → 性能瓶颈。

### 6.2 非事件驱动架构 (Confidence 10/10)
当前系统为批处理模式。生产需要：事件循环（DataEvent → SignalEvent → OrderEvent → FillEvent）、异步消息传递。

### 6.3 单进程故障域 (Confidence 10/10)
所有模块运行在同一进程。XGBoost OOM → 全系统崩溃。akshare 超时 → 全系统阻塞。无进程监护。
